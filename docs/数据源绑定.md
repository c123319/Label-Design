数据绑定建议按“**数据源 → 字段映射 → 元素绑定 → 预览 → 批量生成**”这条链路设计。不要一开始就只做一个 Excel 上传按钮，否则后面批量生成、模板复用、字段校验都会很难维护。

下面是一套比较完整、可落地的设计。

---

# 一、数据绑定整体目标

数据绑定的作用是：让用户设计一次模板，然后通过 Excel / CSV / JSON / API 数据批量生成标签。

例如模板里有：

```text
产品名称：{{productName}}
规格型号：{{spec}}
序列号：{{serialNo}}
生产日期：{{produceDate}}
```

用户上传 Excel：

| productName | spec      | serialNo | produceDate |
| ----------- | --------- | -------- | ----------- |
| 伺服电机        | MP-SM1000 | SN001    | 2024-05-28  |
| 控制模块        | AT-3220C  | SN002    | 2024-05-29  |

系统最终按每一行数据生成一张标签。

---

# 二、核心功能模块

## 1. 数据源管理

数据源是批量生成的基础。

### 支持的数据源类型

建议第一期支持：

| 类型      | 说明         | 优先级 |
| ------- | ---------- | --- |
| Excel   | `.xlsx` 文件 | 高   |
| CSV     | `.csv` 文件  | 高   |
| 手动表格    | 页面中直接录入数据  | 中   |
| JSON    | 开发调试或接口导入  | 中   |
| API 数据源 | 从后端业务系统拉取  | 后期  |

第一期最推荐：**Excel + CSV + 手动表格**。

---

## 2. 数据上传流程

### 用户操作流程

```text
点击“数据绑定”
→ 上传 Excel / CSV
→ 系统解析表头
→ 展示数据预览
→ 用户确认
→ 生成当前模板的数据源
```

### Excel 示例

第一行必须作为字段名：

```text
productName | spec | serialNo | produceDate | inspector
伺服电机     | 220V | SN001    | 2024-05-28  | QC-01
```

### 上传校验

需要校验：

| 校验项  | 规则          |
| ---- | ----------- |
| 文件格式 | xlsx / csv  |
| 文件大小 | 例如不超过 20MB  |
| 表头   | 第一行不能为空     |
| 字段名  | 不允许重复       |
| 数据行  | 至少 1 行      |
| 空字段  | 可提示但不阻止     |
| 特殊字符 | 防止公式注入、脚本内容 |

---

# 三、字段设计

## 1. 字段对象结构

后端解析 Excel 后，应该形成统一字段结构：

```json
{
  "fieldCode": "productName",
  "fieldName": "产品名称",
  "fieldType": "text",
  "required": false,
  "defaultValue": "",
  "sampleValue": "伺服电机"
}
```

### 字段说明

| 字段           | 说明          |
| ------------ | ----------- |
| fieldCode    | 字段编码，内部唯一标识 |
| fieldName    | 显示名称        |
| fieldType    | 字段类型        |
| required     | 是否必填        |
| defaultValue | 默认值         |
| sampleValue  | 示例值         |

---

## 2. 字段类型

建议支持：

| 类型      | 说明    |
| ------- | ----- |
| text    | 普通文本  |
| number  | 数字    |
| date    | 日期    |
| image   | 图片地址  |
| barcode | 条码内容  |
| qrcode  | 二维码内容 |
| enum    | 枚举值   |
| boolean | 是 / 否 |

第一期可以先统一按 `text` 处理，后面再扩展类型。

# 四、模板元素绑定设计

每个画布元素都可以绑定数据字段。

## 1. 文本元素绑定

### 示例

文本内容：

```text
产品名称：{{productName}}
规格型号：{{spec}}
```

渲染结果：

```text
产品名称：伺服电机
规格型号：MP-SM1000
```

### 文本元素数据结构

```json
{
  "id": "el_text_001",
  "type": "text",
  "content": "产品名称：{{productName}}",
  "bindings": [
    {
      "fieldCode": "productName",
      "placeholder": "{{productName}}"
    }
  ],
  "style": {
    "fontSize": 14,
    "fontFamily": "Microsoft YaHei",
    "color": "#000000"
  },
  "position": {
    "x": 10,
    "y": 20,
    "width": 80,
    "height": 8
  }
}
```

---

## 2. 条码元素绑定

条码不是简单文本，它要绑定“条码内容字段”。

例如：

```json
{
  "id": "el_barcode_001",
  "type": "barcode",
  "barcodeType": "CODE128",
  "value": "{{serialNo}}",
  "binding": {
    "fieldCode": "serialNo"
  },
  "showText": true,
  "position": {
    "x": 20,
    "y": 45,
    "width": 60,
    "height": 15
  }
}
```

渲染时：

```text
{{serialNo}} → SN202405280001
```

然后后端/前端根据该值生成条码图片。

---

## 3. 二维码元素绑定

二维码可以绑定单字段，也可以绑定多个字段拼接。

### 单字段绑定

```json
{
  "type": "qrcode",
  "value": "{{serialNo}}",
  "binding": {
    "fieldCode": "serialNo"
  }
}
```

### 多字段组合绑定

```json
{
  "type": "qrcode",
  "value": "产品：{{productName}}\n型号：{{spec}}\n序列号：{{serialNo}}",
  "bindings": [
    { "fieldCode": "productName" },
    { "fieldCode": "spec" },
    { "fieldCode": "serialNo" }
  ]
}
```

---

## 4. 图片元素绑定

适用于批量生成不同图片，例如产品图、Logo、头像、二维码背景图。

```json
{
  "id": "el_image_001",
  "type": "image",
  "src": "{{productImage}}",
  "binding": {
    "fieldCode": "productImage"
  }
}
```

Excel 中可以放：

```text
productImage
https://xxx.com/a.png
/local/file/a.png
```

第一期如果图片绑定复杂，可以先不做，或者仅支持固定图片。

# 六、批量生成流程

## 1. 用户流程

```text
设计模板
→ 上传数据
→ 设置字段绑定
→ 预览首条数据
→ 校验所有数据
→ 提交批量生成
→ 后端异步生成
→ 下载 PDF / ZIP
```

---

## 2. 生成前校验

提交生成前必须校验：

| 校验项        | 说明            |
| ---------- | ------------- |
| 模板是否保存     | 未保存则先保存       |
| 是否有数据源     | 没有数据源不能批量     |
| 占位符是否都匹配字段 | 找不到字段要提示      |
| 条码字段是否为空   | 空值会导致生成失败     |
| 二维码字段是否为空  | 可允许但要提示       |
| 数据总量是否超限   | 例如一次最多 5000 条 |
| 图片字段是否可访问  | 后期支持          |

---

## 3. 异步生成

批量生成不要同步接口阻塞，必须做任务。

```text
POST /api/render/jobs
→ 返回 jobId
→ 前端轮询进度
→ 完成后下载
```

任务状态：

```json
{
  "jobId": "job_20240609_001",
  "status": "PROCESSING",
  "total": 1000,
  "success": 235,
  "failed": 0,
  "progress": 23.5,
  "downloadUrl": null
}
```

# 十、占位符解析规则

## 1. 占位符格式

统一使用：

```text
{{fieldCode}}
```

例如：

```text
{{productName}}
{{serialNo}}
{{produceDate}}
```

---

## 2. 替换函数逻辑

```js
function renderText(template, rowData) {
  return template.replace(/\{\{([\w.-]+)\}\}/g, (_, fieldCode) => {
    const value = rowData[fieldCode]
    return value === undefined || value === null ? '' : String(value)
  })
}
```

---

## 3. 默认值支持

可以扩展：

```text
{{productName|默认产品}}
```

解析逻辑：

```text
如果 productName 为空，则显示“默认产品”
```

---

## 4. 格式化支持

后期可以支持：

```text
{{produceDate|date:yyyy-MM-dd}}
{{price|number:0.00}}
{{serialNo|upper}}
```

第一期不建议做太复杂，先做基础替换。

# 十二、前端预览渲染逻辑

切换到预览模式时：

```js
function renderPreview(objects, rowData) {
  return objects.map(obj => {
    const newObj = structuredClone(obj)

    if (obj.type === 'text') {
      newObj.content = renderText(obj.contentTemplate || obj.content, rowData)
    }

    if (obj.type === 'barcode') {
      newObj.value = renderText(obj.valueTemplate || obj.value, rowData)
      newObj.image = generateBarcode(newObj.value, obj.barcodeType)
    }

    if (obj.type === 'qrcode') {
      newObj.value = renderText(obj.valueTemplate || obj.value, rowData)
      newObj.image = generateQRCode(newObj.value)
    }

    return newObj
  })
}
```

注意：
预览模式下不要覆盖原始模板内容，要生成一份临时渲染对象。

# 十三、后端生成逻辑

后端批量生成时不能依赖前端预览结果，必须自己按模板和数据重新渲染。

伪代码：

```java
for (RowData row : dataRows) {
    Template templateCopy = clone(template);

    for (Element element : templateCopy.getObjects()) {
        if ("text".equals(element.getType())) {
            element.setContent(renderText(element.getContentTemplate(), row));
        }

        if ("barcode".equals(element.getType())) {
            String value = renderText(element.getValueTemplate(), row);
            element.setBarcodeImage(generateBarcode(value));
        }

        if ("qrcode".equals(element.getType())) {
            String value = renderText(element.getValueTemplate(), row);
            element.setQrImage(generateQrCode(value));
        }
    }

    renderToPdfOrImage(templateCopy);
}
```

# 十五、最终推荐设计

你这个系统的数据绑定第一期可以这么做：

```text
1. 先让用户上传 Excel；
2. 后端解析表头和前 10 行数据；
3. 前端右侧显示字段列表；
4. 用户选中文本/条码/二维码元素；
5. 在右侧选择绑定字段；
6. 文本用 {{field}} 替换；
7. 条码/二维码用字段值生成；
8. 提供“预览第 N 条数据”；
9. 点击“批量生成”；
10. 后端异步任务生成 PDF 或 ZIP。
```
