# 二、核心文件说明

## 1. `manifest.json`

* 存储模板列表，前端加载模板卡片时直接用
* 示例：

```json
{
  "version": "2026-06-09.001",
  "templates": [
    {
      "id": "general_100x100",
      "name": "通用标签模板",
      "category": "general",
      "categoryName": "通用",
      "width": 100,
      "height": 100,
      "unit": "mm",
      "thumbnail": "thumbnails/general_100x100.png",
      "templateUrl": "templates/general_100x100.json",
      "tags": ["通用", "合格证"],
      "featured": true,
      "sort": 100,
      "version": "1.0.0"
    }
  ]
}
```

---

## 2. `categories.json`

* 管理模板分类，前端显示分类下拉或折叠面板
* 示例：

```json
[
  {"code": "featured", "name": "精选"},
  {"code": "general", "name": "通用"},
  {"code": "logistics", "name": "物流"},
  {"code": "product", "name": "商品"},
  {"code": "food", "name": "食品"},
  {"code": "crossborder", "name": "跨境"},
  {"code": "warehouse", "name": "仓储"}
]
```

---

## 3. `templates/*.json`

* 存储模板内容及元素绑定
* 示例（标签 JSON）：

```json
{
  "schemaVersion": "1.0",
  "templateId": "general_100x100",
  "name": "通用标签模板",
  "category": "general",
  "canvas": {
    "width": 100,
    "height": 100,
    "unit": "mm",
    "background": "#FFFFFF",
    "gridSize": 2,
    "showGrid": true
  },
  "objects": [
    {
      "id": "el_text_title",
      "type": "text",
      "x": 10,
      "y": 10,
      "width": 80,
      "height": 8,
      "content": "产品名称：{{productName}}",
      "style": {"fontSize": 14, "fontWeight": "bold", "color": "#000000"},
      "binding": {"enabled": true, "fields": ["productName"]}
    },
    {
      "id": "el_barcode_sn",
      "type": "barcode",
      "barcodeType": "CODE128",
      "x": 10,
      "y": 40,
      "width": 70,
      "height": 15,
      "value": "{{serialNo}}",
      "showText": true,
      "binding": {"enabled": true, "fieldCode": "serialNo"}
    }
  ]
}
```

---

## 4. `assets/`

* 存放模板可复用的素材，如 logo、图标、背景图片等
* 前端模板 JSON 可以直接引用 URL，例如：
  `"src": "/assets/logos/company_logo.png"`

# 三、前端加载逻辑

1. 配置仓库地址（GitHub / Gitee / OSS）

```js
const TEMPLATE_STORE_URL = 'https://raw.githubusercontent.com/xxx/template-store/main/manifest.json';
```

2. 加载 `manifest.json` 获取模板列表
3. 加载 `categories.json` 获取分类信息
4. 点击模板卡片时加载模板 JSON (`templateUrl`) 并渲染画布
5. 缩略图直接用 `thumbnail` 字段加载

# 四、实施计划

| 阶段   | 内容                         | 负责人   | 备注                                    |
| ---- | -------------------------- | ----- | ------------------------------------- |
| 阶段 1 | 搭建 `template-store` 仓库基础结构 | 前端/产品 | 包含 manifest、分类、部分模板 JSON、缩略图          |

| 阶段 6 | 后期可接后端                     | 后端    | 后端模板中心替代 GitHub / OSS，可管理权限、版本和审核     |

# 五、注意事项

1. **性能优化**

   * manifest.json 只存列表信息，模板 JSON 单独请求
   * 缩略图尽量压缩，不要一次性加载大图

2. **版本控制**

   * manifest.json 带 `version`，前端可缓存
   * 模板 JSON 带 `schemaVersion`，保证前端兼容

3. **安全**

   * 禁止模板 JSON 包含脚本
   * 所有图片 / URL 都要校验合法域名或 OSS 地址

4. **扩展性**

   * 后期接后端只需要改 `TEMPLATE_STORE_URL` 对应接口即可
   * 支持多仓库切换、用户自定义模板上传
